Goal

In this exercise you will transform the game application into a real-time application, where the content is dynamically loaded and rendered directly in the browser and there is no need to refresh.

This will further enhance the user experience for the following features:

  • synchronizing the editing of typing challenges
  • implementing the multi-player version of one of the games (your choice)

To do so, we will build upon the JSON API and the single-page application from the previous assignments. Also this time, you may and should run JavaScript code both in the browser and on the Web server.

Developer Productivity Hint: do not start solving this assignment from scratch, but reuse as much as possible the code from your solution of assignments 1,2,3,4,5,6,7.

To make testing easier, we recommend solving this assignment in pairs.

Task 1. Synchronize Editing Challenges

All state changes to the typing challenges should be reflected on all connected browser without neither:

  • having to manually refresh the browser page
  • automatically polling the server for updates

Creating or updating a challenge from one browser should automatically refresh the challenge listed on all browsers.

Deleting an existing challenge from one browser should automatically remove it from all browsers displaying the challenge list.

Task 2. ★☆☆ Multiplayer tic-tac-toe game

A pair of players can play the game against each other and anyone else can watch them play from their browser.

When players make their move by clicking on a link, all connected browsers should "navigate" to the same game state.

Do not reload the page when clicking on a link, but replace the current board with the one corresponding to the link URL.

Only display clickable links on the browser of the corresponding player.

Task 3. ★★☆ Multiplayer typing game

Implement the following use case scenarios:

A) Start game

All players should be ready to start playing their multiplayer match at the same time.

When they are ready, they should click on the start button.

Display the number of connected players and how many have clicked on the start button so far.

Start the game only after all players have agreed to start.

The same randomly chosen typing challenge should be displayed on all browsers.

B) Track typing race progress

As the players make progress during their games, display the typed string (showing whether the characters are correct, incorrect or not yet entered) of each player on all connected browsers.

C) Game over

Wait for the slowest player to finish typing the string.

Display the score and elapsed time for all players across all browsers, indicating the winner (highest score, lowest time)

Task 4. ★★★ Multiplayer snake game

Implement the following use case scenarios:

A) Start game

Players can join and leave the game at any time.

Display the number of connected players and keep it up-to-date if additional players join.

B) Synchronize Snakes

The position, direction, color of the snakes should be the same across all browsers.

Display the initials of the player name next to the snake head.

Synchronize the snake object states only when the user changes its direction or when the snake length grows because it has encountered a food item.

When a snake encounters another snake, the snake biting the other snake should increase its health. The other should decrease its health. However, if both snake heads overlap, both should die immediately.

C) Synchronize Food

The food objects should be placed at the same positions and be shown with the same size and color across all browsers.

When a food object is eaten by a snake it should disappear from all browsers and the snake length should grow by the same amount on all browsers.

D) Game over

The game stops on all browsers as soon as the health of any snake reaches 0%.

Display the scores and the snake colors of all players indicating the winner (highest score) on the game over view.
